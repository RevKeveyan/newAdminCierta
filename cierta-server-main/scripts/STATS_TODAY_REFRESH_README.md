# Statistics Today Refresh - Обновление статистики за сегодня

## Описание

Cron job для периодического обновления статистики за сегодняшний день. Необходимо для актуальных данных, особенно для expired loads, которые зависят от текущего времени.

## Расположение

`server/cron/statsTodayRefreshCron.js`

## Как работает

### Алгоритм

1. **Запуск**: Каждые 1-5 минут (настраивается через `REFRESH_INTERVAL_MINUTES`)
2. **Действие**: Помечает сегодняшний день как "грязный" для system статистики
3. **Источники**: `['loads', 'receivable', 'payable']`
4. **Обработка**: Воркер (`statsWorker`) автоматически обработает задачу и обновит snapshot

### Код

```javascript
markDirtyDay(today, 'system', null, ['loads', 'receivable', 'payable'])
```

## Конфигурация

### Интервал обновления

По умолчанию: **3 минуты**

Изменить можно в `statsTodayRefreshCron.js`:

```javascript
const REFRESH_INTERVAL_MINUTES = 3; // Изменить на 1-5 минут
```

### Cron pattern

Текущий pattern: `*/3 * * * *` (каждые 3 минуты)

Для других интервалов:
- `*/1 * * * *` - каждую минуту
- `*/2 * * * *` - каждые 2 минуты
- `*/5 * * * *` - каждые 5 минут

## Запуск

Cron запускается автоматически при старте сервера через `app.js`:

```javascript
require("./cron/statsTodayRefreshCron");
```

Также выполняется сразу при старте (не ждёт первого cron запуска).

## Зачем это нужно

### Expired Loads

Expired loads зависят от текущего времени (`dates.deadlineAt < now`). Без периодического обновления:
- Новые expired loads не будут отображаться в статистике
- Статистика будет устаревшей

### Актуальные данные

- Новые loads создаются в реальном времени
- Payments обновляются в реальном времени
- Статистика должна отражать текущее состояние

### Производительность

- Обновляется только один день (сегодня)
- Только system уровень (не все customers/carriers)
- Воркер обрабатывает задачу асинхронно
- Минимальная нагрузка на систему

## Мониторинг

Проверить работу cron:

```javascript
// Проверить последние dirty задачи для сегодня
const today = getDateKeyUTC5();
const dirtyTasks = await StatsDirty.find({
  dateKey: today,
  entityType: 'system',
  entityId: null
}).sort({ createdAt: -1 }).limit(5);

// Проверить последний snapshot для сегодня
const snapshot = await StatsSnapshot.findOne({
  grain: 'day',
  dateKey: today,
  entityType: 'system',
  entityId: null
}).sort({ computedAt: -1 });
```

## Альтернативные подходы

### Отдельный endpoint "expired live"

Можно было бы создать отдельный endpoint для expired loads, который вычисляет их в реальном времени:

```javascript
GET /stats/expired-live?entityType=system&entityId=...
```

**Плюсы:**
- Всегда актуальные данные
- Не требует периодического обновления

**Минусы:**
- Усложняет API
- Требует отдельной логики
- Медленнее (вычисление на лету)

**Решение:** Использовать периодическое обновление (проще и эффективнее)

## Рекомендации

1. **Интервал 3-5 минут** - оптимальный баланс между актуальностью и нагрузкой
2. **Только system** - не обновлять customer/carrier статистику (слишком тяжело)
3. **Мониторинг** - следить за временем обработки задач воркером
4. **Логирование** - проверять логи на наличие ошибок

## Troubleshooting

### Cron не запускается

Проверить:
- Запущен ли сервер
- Есть ли ошибки в логах при старте
- Правильно ли настроен cron pattern

### Статистика не обновляется

Проверить:
- Работает ли воркер (`statsWorkerCron`)
- Есть ли задачи в `StatsDirty` для сегодня
- Обрабатываются ли задачи воркером

### Слишком частое обновление

Если нагрузка высокая:
- Увеличить `REFRESH_INTERVAL_MINUTES` до 5 минут
- Проверить производительность воркера
- Оптимизировать запросы в `computeSnapshot`
