# Система обновления статистики - Интервалы и периоды

## Обзор архитектуры

Система статистики работает по принципу **"dirty queue"** (очередь грязных данных):
1. При изменении данных создаются задачи в `StatsDirty`
2. Воркер обрабатывает задачи из очереди
3. Результаты сохраняются в `StatsSnapshot`

## Компоненты системы

### 1. StatsWorker (Воркер обработки)

**Файл**: `server/services/statsWorker.js`

**Интервал обработки**: **5 секунд** (5000ms)

**Как работает**:
- Проверяет очередь `StatsDirty` каждые 5 секунд
- Обрабатывает до 10 задач за раз (`batchSize: 10`)
- Блокирует задачу перед обработкой (lock mechanism)
- Вычисляет snapshot для диапазона дат
- Сохраняет результат в `StatsSnapshot`
- Удаляет задачу из очереди после успешной обработки

**Настройки**:
```javascript
{
  interval: 5000,    // Проверка каждые 5 секунд
  batchSize: 10      // До 10 задач за раз
}
```

**Запуск**: Автоматически при старте сервера через `statsWorkerCron.js`

### 2. StatsTodayRefreshCron (Обновление сегодняшнего дня)

**Файл**: `server/cron/statsTodayRefreshCron.js`

**Интервал**: **3 минуты** (`*/3 * * * *`)

**Что делает**:
- Помечает сегодняшний день как "dirty" для system статистики
- Источники: `['loads', 'receivable', 'payable']`
- Необходимо для актуальных данных, особенно для expired loads

**Настройка**:
```javascript
const REFRESH_INTERVAL_MINUTES = 3; // Можно изменить на 1-5 минут
```

**Cron pattern**: `*/3 * * * *` (каждые 3 минуты)

**Запуск**: 
- Автоматически при старте сервера
- Выполняется сразу при старте (не ждёт первого cron запуска)

### 3. Автоматическое помечание dirty при изменениях

**Файл**: `server/utils/markDirty.js`

**Когда срабатывает**:
- При создании/обновлении/удалении Load → `markDirtyForLoad()`
- При создании/обновлении/удалении Payment → `markDirtyForPayment()`

**Что помечается**:
- `system` статистика (всегда)
- `customer` статистика (если есть customer)
- `carrier` статистика (если есть carrier)
- `user` статистика (если есть createdBy)

**Grain**: Всегда **`day`** (только дневные snapshots)

## Периоды (Grain) и их обработка

### Day (День)

**Grain**: `day`
**DateKey формат**: `YYYY-MM-DD` (например, `2026-02-12`)

**Обновление**:
- ✅ Автоматически при изменениях данных
- ✅ Каждые 3 минуты для сегодняшнего дня (через cron)
- ✅ Обрабатывается воркером каждые 5 секунд

**Хранение**: Отдельный snapshot для каждого дня

### Week (Неделя)

**Grain**: `week`
**DateKey формат**: `YYYY-WNN` (например, `2026-W07`)

**Обновление**:
- ⚠️ **НЕ обновляется автоматически**
- ⚠️ Вычисляется **на лету** из day snapshots при запросе API
- ⚠️ Нет отдельных задач в StatsDirty для week

**Как работает**:
- При запросе `/stats/loads?grain=week` API ищет week snapshots
- Если week snapshot отсутствует, возвращается пустой snapshot
- Для вычисления week статистики нужно агрегировать day snapshots (но это не реализовано автоматически)

### Month (Месяц)

**Grain**: `month`
**DateKey формат**: `YYYY-MM` (например, `2026-02`)

**Обновление**:
- ⚠️ **НЕ обновляется автоматически**
- ⚠️ Вычисляется **на лету** из day snapshots при запросе API
- ⚠️ Нет отдельных задач в StatsDirty для month

**Как работает**:
- При запросе `/stats/loads?grain=month` API ищет month snapshots
- Если month snapshot отсутствует, возвращается пустой snapshot
- Для вычисления month статистики нужно агрегировать day snapshots (но это не реализовано автоматически)

### Year (Год)

**Grain**: `year`
**DateKey формат**: `YYYY` (например, `2026`)

**Обновление**:
- ⚠️ **НЕ обновляется автоматически**
- ⚠️ Вычисляется **на лету** из day snapshots при запросе API
- ⚠️ Нет отдельных задач в StatsDirty для year

**Как работает**:
- При запросе `/stats/loads?grain=year` API ищет year snapshots
- Если year snapshot отсутствует, возвращается пустой snapshot
- Для вычисления year статистики нужно агрегировать day snapshots (но это не реализовано автоматически)

## Текущее состояние системы

### ✅ Что работает автоматически:

1. **Day snapshots**:
   - Обновляются при каждом изменении данных
   - Обновляются каждые 3 минуты для сегодняшнего дня
   - Обрабатываются воркером каждые 5 секунд

2. **Автоматическое помечание dirty**:
   - Load create/update/delete → помечает соответствующие дни
   - Payment create/update/delete → помечает соответствующие дни
   - Помечаются все связанные сущности (system, customer, carrier, user)

### ⚠️ Что НЕ работает автоматически:

1. **Week/Month/Year snapshots**:
   - НЕ создаются автоматически при изменении данных
   - НЕ обновляются через cron
   - API возвращает пустые snapshots, если они не существуют

2. **Агрегация из day snapshots**:
   - НЕ реализована автоматическая агрегация week/month/year из day snapshots
   - При запросе с `grain=month` API ищет month snapshot, но не вычисляет его из day snapshots

## Как это работает сейчас

### Сценарий 1: Запрос day статистики

```
GET /stats/loads?from=2026-02-01&to=2026-02-12&grain=day
```

1. API генерирует dateKeys: `['2026-02-01', '2026-02-02', ..., '2026-02-12']`
2. Ищет snapshots с `grain='day'` и этими dateKeys
3. Возвращает найденные snapshots (или пустые, если отсутствуют)

### Сценарий 2: Запрос month статистики

```
GET /stats/loads?from=2026-02-01&to=2026-02-28&grain=month
```

1. API генерирует dateKeys: `['2026-02']`
2. Ищет snapshots с `grain='month'` и dateKey `'2026-02'`
3. Если snapshot отсутствует → возвращает пустой snapshot
4. **НЕ агрегирует** из day snapshots автоматически

## Рекомендации для улучшения

### Вариант 1: Агрегация на лету (рекомендуется)

Модифицировать `StatsController.getSnapshotsForDateKeys()`:
- Если snapshot с нужным grain отсутствует
- Агрегировать из day snapshots для этого периода
- Сохранить результат в StatsSnapshot для кэширования

### Вариант 2: Отдельные задачи для week/month/year

Создать функции `markDirtyWeek()`, `markDirtyMonth()`, `markDirtyYear()`:
- При изменении day snapshot помечать соответствующие week/month/year как dirty
- Воркер будет обрабатывать эти задачи
- Вычислять week/month/year snapshots из day snapshots

### Вариант 3: Cron для агрегации

Создать cron задачи:
- Ежедневно агрегировать week snapshots из day snapshots
- Ежемесячно агрегировать month snapshots из day snapshots
- Ежегодно агрегировать year snapshots из day snapshots
